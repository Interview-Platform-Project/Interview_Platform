# Interview_Platform

## Branch name pattern

feat/ — новая функциональность / страница / компонент.

update/ — доработка или обновление функциональности / страницы / компонента.

fix/ — исправление ошибки.

docs/ — изменения в документации.

style/ — изменения в стиле кода (форматирование, пробелы и т.д.).

refactor/ — рефакторинг кода без изменения функционала.

chore/ — задачи по обслуживанию проекта (обновления зависимостей, настройка окружения и т.д.).

### Прммер
feat/user-authentication;

fix/login-page-error;

## Commit name pattern

<тип>[контекст]: <описание> [необязательное тело]

Feat - создание / добавление новой фичи / функции / класса / файла.

Update - обновление / дополнение фичи / функции / класса / файла.

Fix - исправлен уже обнаруженный баг / ошибка или исправлен код, который мог привести к ошибке в следствии ранее внесённых изменений.

VIP - "version in progress" используя этот тип у вас появляется возможность сделать коммит в любой момент в случае если вам нужно срочно прервать работу над текущей задачей и вы не хотели бы потерять изменения. Примеры: закончился рабочий день, вам нужен отдых, вас выдёргивают на другой проект или срочный fix. Нет ничего критичного даже в отправке этого коммита в удаленный репозиторий - вы сообщили другим разработчикам что вы еще в процессе реализации.

Refactor - рефакторинг без изменения ожидаемого результата. Например, вы просто переписали функцию в декларативном стиле при этом возвращаемое значение или суть работы функции, не изменились.

Docs - добавлена или изменена документация.

Style – изменения в стиле кода (форматирование, пробелы и т.д.). Да, форматирование кода – отдельный коммит, а работа над фичей, или fix – отдельный коммит.

Chore - задачи по обслуживанию проекта (обновления зависимостей, настройка окружения и т.д.).
