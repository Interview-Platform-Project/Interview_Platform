# Interview Platform

Учебный проект: сервис проведения технических и мок-интервью (realtime-редактор
кода, видео, AI-подсказки, витрина участников, обратная связь, лидерборд,
уведомления в Telegram).

> Это **скелет**. Здесь намеренно нет стека и конфигов — только направление.
> Каждая команда поднимает своё приложение с нуля и наслаивает инструменты по
> мере разработки.

## Структура

```
interview-platform/
├─ apps/
│  ├─ backend/          # серверное приложение (поднимает команда бэкенда)
│  └─ frontend/         # клиентское приложение (поднимает команда фронтенда)
├─ packages/
│  └─ shared/           # общий код (типы/контракты/константы) — пока пустой
├─ package.json         # корень монорепы
└─ pnpm-workspace.yaml  # состав воркспейса (apps/*, packages/*)
```

## Монорепа

Управление пакетами — **pnpm** (см. `pnpm-workspace.yaml`). Приложения живут в
`apps/*`, переиспользуемый код — в `packages/*`.

## С чего начать команде

1. Определиться со стеком своего приложения (`apps/backend` / `apps/frontend`).
2. Инициализировать проект внутри своей папки, добавить зависимости.
3. По мере необходимости наслаивать общее: линтеры, форматтер, тесты, Docker,
   CI/CD, observability и т.д.
4. Общие типы/контракты между бэком и фронтом класть в `packages/shared`.

Договорённости команды (git-флоу, код-ревью, созвоны, доска) фиксируются
отдельно по мере старта.
